Detailed Topics — Hinglish explanations + Important Commands

Yeh section short aur clear Hinglish mein har important topic ka long description aur commonly used Cisco commands (what / why / where / expected result) deta hai. Copy kar ke Packet Tracer mein practice karo.

Basics — Packet Tracer (PT)

PT ek simulation tool hai jisme aap virtual devices (routers, switches, PCs) connect karke networking practise karte ho. Real hardware ki tarah CLI aur GUI dono available hain.

Commands / Steps:
  • File → New — naya topology banaane ke liye.
  • Save / File → Save As — topology save karna (important).
  • Simulation mode: packet ko inspect karke OSI layer wise dekho — expected: packet details open hoti hain.

Cisco CLI Basics — Kaliye simple

CLI modes: User EXEC (>), Privileged EXEC (#), Global Config (config)#, Interface config (config-if)#. Har mode ka purpose alag — read-only / diagnostic se lekar configuration tak.

Important commands (what | why | where | result):
  • enable — kya: user exec se privileged exec mode me jaate; kyon: kuch commands sirf # mode me chalti; expected: prompt changes from Router> to Router#.
  • configure terminal (conf t) — kya: global config mode; kyon: interfaces/config options set karne ke liye; expected: prompt Router(config)#.
  • hostname R1 — kya: device name set; kyon: topology me identify karna; expected: prompt shows new name.
  • interface fa0/0 — kya: specific interface pe jao; kyon: IP assign, shutdown/no shutdown; expected: prompt Router(config-if)#.
  • ip address 192.168.1.1 255.255.255.0 — kya: IP set; kyon: device ko network me communicate karana; expected: interface up if connected correctly.
  • no shutdown — kya: interface enable; kyon: by default some interfaces shutdown ho sakte; expected: interface up line protocol up (check show interfaces).
  • copy running-config startup-config — kya: save current config to NVRAM; kyon: reboot ke baad settings chahiye; expected: confirmation and saved message.
  • show running-config — kya: current config dekhna; kyon: troubleshoot and verify; expected: full running configuration shows.
  • show ip interface brief — kya: quick status of interfaces; kyon: fast troubleshoot; expected: list of interfaces, IPs, and up/down.

Subnetting (Hinglish)

Subnetting ka matlab IP range ko chote-chote networks me divide karna. Binary samajhna zaroori hai: host bits aur network bits alag kar ke subnet mask banta.

Examples & Commands:
  • Practice: convert /24 = 255.255.255.0 (256 addresses, 254 usable). /26 = 255.255.255.192 (64 addresses, 62 usable).
  • Packet Tracer: assign IP on PC ya interface: ip address 10.0.0.1 255.255.255.192 (interface config mode). Expected: correct network connectivity within subnet.

VLANs & Switching

VLAN = Virtual LAN. Ek physical switch me multiple logical networks banate ho. Trunking se multiple VLANs ek link se travel kar sakte (802.1Q tagging).

Important switch commands (what | why | where | result):
  • enable → conf t — enter config mode.
  • vlan 10 — kya: VLAN create karna; expected: VLAN created in vlan database.
  • name Sales — kya: VLAN name set; expected: visible in show vlan brief.
  • interface range fa0/1 - 2 / switchport access vlan 10 — kya: ports ko VLAN assign; expected: PC connected to that port gets in same VLAN (if IPs set).
  • interface fa0/24 / switchport trunk encapsulation dot1q / switchport mode trunk — kya: trunk port configure; expected: trunk shows in show interfaces trunk.
  • show vlan brief & show interfaces trunk — verify configuration.

Inter‑VLAN Routing (Router-on-a-stick)

Jab VLANs alag networks ho to unko communicate karane ke liye L3 device chahiye. Router-on-a-stick me router ke subinterfaces bana ke 802.1Q tagging use karte hain.

Commands (router):
  • interface g0/0.10 — kya: subinterface; kyon: VLAN 10 traffic tag karne; expected: new logical interface.
  • encapsulation dot1Q 10 — kya: VLAN tag set; expected: router accepts tagged frames for VLAN 10.
  • ip address 192.168.10.1 255.255.255.0 — kya: SVI IP for that VLAN; expected: default gateway for VLAN clients.
  • show ip route / show ip interface brief — verify reachability.

Spanning Tree Protocol (STP)

STP loops ko rokta hai by blocking certain ports so single active path rahe. Important jab multiple switch links ho.

Commands & checks:
  • show spanning-tree — kya: STP state dekho; expected: root bridge, port roles (root/alt/designated).
  • spanning-tree vlan 1 priority 4096 — kya: priority lower karoge to root banega; expected: root bridge change after convergence.

Routing — Static & OSPF (Hinglish)

Static routing simple hai (manual). OSPF dynamic routing protocol hai jo networks ko automatically routes share kar ke path choose karta.

Static commands:
  • ip route 10.0.2.0 255.255.255.0 192.168.1.2 — kya: static route add; kyon: jab next-hop fixed ho; expected: route appears in show ip route.
OSPF basics commands:
  • router ospf 1 — kya: enable OSPF process 1; expected: process running in router.
  • network 192.168.1.0 0.0.0.255 area 0 — kya: advertise network into OSPF area 0; expected: neighbors form, routes learned.
  • show ip ospf neighbor — kya: OSPF neighbors; expected: list of adjacent routers and states.

NAT (PAT) & ACLs — Internet access & filtering

NAT private IPs ko public IP me translate karta (internet access). ACLs traffic ko allow/deny karte — security ka basic tool.

NAT (PAT) example:
  • access-list 1 permit 192.168.10.0 0.0.0.255 — define inside range for NAT.
  • ip nat pool MYPOOL 203.0.113.10 203.0.113.10 netmask 255.255.255.0 — kya: public pool (example).
  • ip nat inside source list 1 interface g0/0 overload — kya: PAT on interface; expected: outbound clients share single public IP.
ACLs example:
  • access-list 100 permit tcp any any eq 80 — kya: permit HTTP; expected: web traffic allowed.
  • interface g0/1 / ip access-group 100 in — apply ACL; expected: filtered traffic per ACL rules.

DHCP & DNS

DHCP automatic IP assign karta. DNS naam ko IP me convert karta. PT me dono services as servers available hain.

DHCP commands (router as DHCP server):
  • ip dhcp pool VLAN10 — kya: create DHCP pool; expected: pool created.
  • network 192.168.10.0 255.255.255.0 — kya: pool network; expected: clients in this network get IPs from pool.
  • default-router 192.168.10.1, dns-server 8.8.8.8 — kya: options to give to clients.
  • show ip dhcp binding — kya: view assigned leases; expected: list of MAC-to-IP leases.
DNS (PT Server):
  • Use PT Server device → Services tab → DNS → Add A records (e.g., www.example.com → 203.0.113.5). Expected: clients can ping hostname and resolve to IP.

Troubleshooting — Practical

Common commands aur unka use: jaldi problem locate karne ke liye yeh must-know.

  • ping <ip> — connectivity test; expected: replies or timeouts.
  • traceroute <ip> — path check; expected: list of hops to destination.
  • show ip interface brief — interface status (up/down). Expected: identify down interfaces.
  • show ip route — routing table; expected: verify learned/static routes.
  • debug ip route (careful in real devices) — kya: live route updates; kyon: deep troubleshooting; expected: verbose logs.

Security fundamentals

SSH, user management aur basic hardening commands. Packet Tracer me ye concepts practice karo lekin real firewalls bhi seekho.

  • username admin privilege 15 secret MyP@ss — local user create; expected: user stored in running-config.
  • line vty 0 4 / transport input ssh / login local — enable SSH login only; expected: telnet disabled if transport input restricts.
  • service password-encryption — simple obfuscation for plain-text passwords in config.

Labs & Projects — Tips

Har lab ka goal clear rakho: objective, topology diagram, step-by-step commands, expected verification commands (show/ ping/ traceroute). Document har lab.

Verification commands:
  • show ip interface brief, show ip route, show running-config, show arp, show mac address-table.
